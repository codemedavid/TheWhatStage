Full Design Summary — Campaign Flow Builder

Everything we're building, in one view. Approve to proceed to implementation plan.

What We're Building

Database (5 new tables)

  • campaigns — name, goal, is_primary, status, follow-up config
  • campaign_phases — replaces bot_flow_phases, scoped to a campaign
  • lead_campaign_assignments — permanent sticky assignment per lead
  • experiments — A/B test groups with start/end/winner
  • experiment_campaigns — weighted join of campaigns to experiments
  • campaign_conversions — conversion event log per campaign

Modified: conversation_phases + exited_at, exit_reason

New UI Pages

  • /app/campaigns — campaign list with conversion rates
  • /app/campaigns/new — create campaign
  • /app/campaigns/[id] — editor (Flow / Settings / Metrics tabs)
  • /app/campaigns/experiments — A/B test manager
  • /app/campaigns/experiments/new — create experiment
  • /app/campaigns/experiments/[id] — live experiment view

API Routes

  • /api/campaigns — CRUD
  • /api/campaigns/[id]/phases — phase CRUD + reorder
  • /api/campaigns/[id]/metrics — phase funnel + conversion stats
  • /api/experiments — CRUD
  • /api/experiments/[id]/promote — promote winner
  • /api/cron/drop-off-scanner — Vercel cron, every 15min

Engine Changes

  • Assign lead to campaign on first message (experiment-aware)
  • Load phases from campaign_phases not bot_flow_phases
  • Track phase exit reason + timestamp
  • Detect conversion via lead_events → log to campaign_conversions
  • Background cron: silence → follow-up → drop-off mark

Migration Path

Existing tenants auto-get a "Default Campaign" (primary) with their current phases copied in. Zero downtime — the engine falls back to the primary campaign if no assignment exists, so old conversations continue uninterrupted during the rollout.

Implementation Phases

Phase 9A Data layer — migrations, types, RLS, migration script for existing phases
Phase 9B Campaign CRUD API + engine wiring — campaigns, phases API, assignment logic, conversion detection
Phase 9C Campaign UI — list, editor (Flow/Settings tabs), nav update, remove old flow builder tab
Phase 9D Experiments UI — experiment manager, A/B assignment, promote-winner flow
Phase 9E Metrics + cron — phase funnel UI, conversion stats, drop-off scanner background job
