Design — Section 4: Conversation Engine Changes

How the engine picks a campaign, assigns a lead, and tracks conversions

Assignment Flow (first message from a lead)

1
Message arrives → look up lead_campaign_assignments by lead_id
2
Assignment found? → load campaign_phases for that campaign. Done — skip to step 5.
3
No assignment: check experiments for status = 'running' for this tenant
→ If running: pick campaign by weighted random (weights from experiment_campaigns)
→ If none: use the tenant's is_primary = true campaign
4
INSERT lead_campaign_assignments — assignment is now permanent
5
Load phases from campaign_phases → run normal conversation engine logic

Conversion Detection

Conversion is triggered by existing lead_events — no new webhook needed. A Supabase trigger (or app-level hook) watches lead_events inserts and fires when the event type matches the campaign goal:

Goal → Event match

'form_submit' → lead_event.type = 'form_submit'
'appointment_booked' → 'appointment_booked'
'purchase' → 'purchase'
'stage_reached' → 'stage_changed'
  + payload.stage_id matches goal_config

On match

1. INSERT campaign_conversions
2. UPDATE conversation_phases
  SET exit_reason = 'converted'
  WHERE open for this lead
3. Bot can acknowledge
  the completed goal

Migration — bot_flow_phases → campaign_phases

① For each tenant that has rows in bot_flow_phases: auto-create a campaign named "Default Campaign" with is_primary = true and goal = tenant's current bot_goal
② Copy all bot_flow_phases rows → campaign_phases, setting campaign_id to the new default campaign
③ Update conversation_phases.phase_id FK to point to the new campaign_phases rows (matched by old ID)
④ Tenants with zero phases get an empty "Default Campaign" — they're prompted to set it up on next login
✓ bot_flow_phases kept as a backup view until migration verified, then dropped

Background Job — Drop-off Scanner

A Next.js cron route (/api/cron/drop-off-scanner) runs every 15 minutes via Vercel Cron:

SELECT cp.*, c.follow_up_delay_minutes, c.follow_up_message,
       cam.id as campaign_id
FROM conversation_phases cp
JOIN conversations conv ON conv.id = cp.conversation_id
JOIN lead_campaign_assignments lca ON lca.lead_id = conv.lead_id
JOIN campaigns cam ON cam.id = lca.campaign_id
WHERE cp.exited_at IS NULL
AND cp.entered_at < now() - (cam.follow_up_delay_minutes * interval '1 minute')
AND last message in conversation < entered_at + delay

→ For each: send follow_up_message via FB Send API
→ After delay*2 with no reply: UPDATE exit_reason = 'dropped', exited_at = now()
