Design Section 4 (Updated): Complete Feature Map

Now with RAG & Bot Training + Goal-Driven Actions — 9 subsystems, 40 features

NEW: 🧠 RAG & Bot Training

Knowledge Base Tenant uploads docs, FAQs, product info, business context — stored and chunked for retrieval
RAG Pipeline Chunk → embed → vector store. Bot retrieves relevant context per conversation turn
Bot Rules & Persona Tenant defines behavior rules ("always ask for email"), tone, boundaries ("never discuss pricing")
Test Conversation Sandbox chat — tenant talks to their own bot, sees RAG context used, tests action buttons
Conversation Review Review past real conversations — flag bad responses, correct them → corrections become rules
Training Dashboard Unified UI — upload docs, set rules, define goal, test, review, iterate

NEW: 🎯 Goal-Driven Actions

Goal Configuration Tenant sets primary goal (qualify, book, sell, collect) — this shapes bot personality and action strategy
Qualification Engine Quiz-style form builder — conditional questions, scoring rules, auto-tag + auto-stage based on answers
Qualification Data View Dashboard view — see every answer a lead gave, their score, which conditions triggered, resulting tags/stage
Booking Integration Bot motivates booking → pushes calendar page → appointment auto-appears on tenant dashboard calendar view
Sales Push Bot showcases products → pushes catalog/checkout → order auto-tracked in commerce dashboard
Action Conditions If/then rules on form answers — e.g. budget > $10k → tag "high-value" → move to "Qualified" → trigger workflow
