Design Section 1: Vault Structure & Note Templates

The folder layout and Markdown templates that both skills will use

Vault Folder Structure

whatstage_obsidian/
📁 Features/ — one note per feature (Lead Pipeline, Workflow Engine...)
📁 Entities/ — one note per DB table (leads, orders, tenants...)
📁 Components/ — one note per UI page or React component
📁 Flows/ — end-to-end user journeys with Mermaid + steps
📁 Index/ — MOCs: System Overview, DB Schema Map, Feature Roadmap, Component Registry

Flow Note Template

Flows/Lead Qualification Flow.md
--- tags: [flow, leads, messenger] subsystem: lead-management created: 2026-04-18 --- # Lead Qualification Flow ## Diagram ```mermaid flowchart LR A[Lead messages on Messenger] --> B[Bot sends action button] B --> C[Lead opens form page] C --> D[Lead submits form] D --> E[lead_event: form_submit] E --> F[AI evaluates context] F --> G[Stage moves to Qualified] G --> H[Notification sent via Messenger] ``` ## Steps 1. **Lead messages on Messenger** → [[Messenger Bot Engine]] receives webhook 2. **Bot sends action button** → [[bot_flows]] config determines which button 3. **Lead opens form page** → [[Action Pages]] serves form at tenant subdomain 4. **Lead submits form** → [[action_submissions]] record created 5. **Event logged** → [[lead_events]] with type `form_submit` 6. **AI evaluates** → HuggingFace model assesses lead context 7. **Stage moves** → [[leads]].stage_id updated to [[stages]] "Qualified" 8. **Notification sent** → [[Messenger Bot Engine]] sends confirmation ## Entities Involved - [[leads]], [[lead_events]], [[action_submissions]], [[stages]], [[bot_flows]] ## Components Involved - [[ActionFormPage]], [[LeadsPage]]
